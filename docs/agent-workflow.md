# AI Agent Workflow

AI can help Stockwise development, but it should not replace product judgment or financial content responsibility.

## Good Uses Of AI

- Product thinking support
- Task decomposition
- Code implementation support
- Code review
- Test idea generation
- Documentation drafts
- Mock official knowledge structure improvement
- Source-grounded helper behavior review

## Human Responsibility

- Product decisions
- MVP scope decisions
- Code understanding
- Final review
- Safety checks
- Financial content responsibility
- Git commits and project direction

## Workflow

1. Read the product docs.
2. Identify the smallest safe task.
3. Propose a short plan.
4. Make minimal scoped changes.
5. Explain changes.
6. Provide manual test steps.
7. Note risks and follow-up tasks.

AI may assist implementation and review, but humans own product decisions, safety checks, and financial content responsibility. For Ask This Note behavior, human review must confirm that answers are grounded only in saved official knowledge and understanding notes.

## AI Task Template

### Goal

What should be accomplished?

### Context

What product or code context matters?

### Scope

What files, behavior, or content may change?

### Non-goals

What should not be changed or added?

### Expected Output

What should exist when the task is complete?

### Manual Test

How should a human confirm the change works?

### Risks

What could go wrong, and what needs review?
