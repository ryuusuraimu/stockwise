# Stockwise Non-Goals

Stockwise should stay focused on safe beginner education through an Investor Knowledge Vault. These items are excluded from the MVP.

## Stock Recommendations

Stockwise should not recommend specific stocks because that would move the product from education into investment advice.

## Investment Advice

Stockwise should explain concepts, not tell users what to buy, sell, or hold.

## Guaranteed Return Claims

No content should imply that returns are certain. Investing always involves risk, including loss of principal.

## Stock Price Prediction

Prediction features can create false confidence and are outside the educational vault purpose.

## Portfolio Management

Portfolio tools add complexity and personal financial context that the MVP does not need.

## Brokerage Integration

Brokerage integration would turn Stockwise toward trading execution, which is outside the learning-focused scope.

## Login

Login is excluded from the MVP to keep the product simple and reduce privacy and implementation complexity.

## User Accounts

User accounts are not needed for validating the local vault, note-saving, and source-grounded learning experience.

## Generic AI Chat

Generic AI chat is outside the MVP because it can blur educational explanations with personalized financial advice.

Ask This Note is allowed only as a constrained mock helper. It must answer from saved official knowledge and understanding notes, not from open-ended model knowledge or external sources.

## Real-Time Market Data

Real-time market data can shift attention from learning concepts to reacting to price movements.

## Complex Backend Database

A complex backend is unnecessary for the MVP and would slow down iteration on the core learning experience.

## Social Sharing

Social sharing is not part of the core learning loop and may encourage shallow engagement over understanding.

## Financial Diagnosis

Stockwise should not evaluate a user's personal financial situation or make personalized recommendations.
